Learning Cycle

23 November 2014

This week as I have gone through more difficult ruby exercises, I learned some enumerables that have helped me solve my code. Enumerables allow searching and sorting in your methods, and they are easy on the eyes if you know what I mean. Maybe you don't. They are easy to read- that's pretty much the point I was trying to get at. This week I came across so many enumerables, but I am going to cover only one of them today.

#Cycle

Cycles allow you to go through an array and cycle through it a specified amount of times. If you don't specify, it goes on forever. Let's take a closer look to see it in action

mult = [1, 2, 3, 4]
mult.cycle { |int| puts int * 2 }

This first piece of code won't do much good except for crashing your computer. In the first part of the code, cycle is called to the mult array. In the brackets, cycle goes through each integer, taking that integer times 2. The first part of the result would look something like this:

=> 2, 4, 6, 8

BUT it doesn't stop there. Cycle will go through it again...and again since we didn't tell it how many times to go through it. So you will be left with a screen full of 2,4,6,8. There is a better use for this, so lets look at how we can do that.

mult.cycle(2) { |int| puts int * 2 }

So there you have it. That is the basics of #cycle!
