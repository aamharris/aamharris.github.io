Relative, Absolute, and Fixed Positionings

9 November 2014

Positions in CSS are kind of a tricky thing to learn, but once you learn how to move things around your design, you will never be the same. Positions give you control over your divs and other elements, so it is crucial that you know how to do it correctly.

Fixed Positioning
We can start out with one of the easier ones: fixed positioning. Can you guess what that does? Yep, it's fixed...doesn't move. This is relative to your browser window. What that means is that even if you have to scroll down the page, that element stays put right where it is. An example that I used this for was my footer at the bottom of this page. I wanted it to be on the page all of the time (so you all would continually be tempted to visit my social media site), so I gave it a fixed position and there it is...all the time.

Relative Positioning

Relative positioning involves moving the object by using the point where it normally sits as a reference point. Normal, or static positioning takes place when you put an element on the page and don't specify where it goes. It just goes after the element you placed before it. For example, if you add a div with text and then put a div with a colorful box, the colorful box would go right below the text. When you use relative positioning and specify left: 20px, it adds 20px to the left and will move the object to the right from its original position. You can also use negative values.

Absolute Positioning
Absolute positioning is a bit tricky. A good rule of thumb for using absolute positioning is to make sure that the parent element you are nested in is using positioning other than static. When you use absolute positioning, you are positioning an object in relation to the parent object. Take a look at this example.

If you look at the code on the left, you see that textbox is a parent of textbox2 (it is nested inside textbox). The position of textbox is not static, so if I set textbox2 to be absolute, it will use the textbox as a reference. Look what happens when you add 50px to the top- it pushes 50px down from the textbox.

When you start practicing postioning, start simple. Make a few simple boxes and practice laying them out where you want. When you start getting good, add some nested items and play around with that. Nothing beats practice!
