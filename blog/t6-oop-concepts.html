Blocs, Procs, and Lambdas, oh my!

7 December 2014

In DBC, we haven't learned about Procs and Lambda's yet, so for the technical blog this week, I decided to dive a little deeper into learning about what these are. If you have worked with ruby even at an early stage, you have worked with blocks, but you might not know much about them. We'll cover that too.

Procs and Lambdas

Lambdas are Proc objects that can be stored in variables. Essentially, lambdas are functions without names and they can return a value just like functions.

For example:

l = lambda {"Hello, World"}
puts l.call

This results in:

"Hello World"

Procs are similar to lambdas, but they do have some differences. If you want to create a proc, you would enter something like this:

proc = Proc.new {"Hello, World"}
puts proc.call

The output of this would be the same as the lambda. But what if we did something like this:

def my_test
proc = Proc.new {return "Hello, World"}
proc.call
return "Good-Bye"
end

When working in ruby, we are used to it returning the last return value. This is not the case with a proc. The output of this would actually return "Hello World".

So Procs return values differently then lambdas. Another difference between the two is that Procs will return 1 argument, even if there are 3 that were passed, whereas lambdas will raise and arugument error.

Blocks

Unlike basically everything in ruby, blocks are not objects. Blocks are part of a method call and if they were to be on their own, you would get an error. When you put something between curly braces (like you would in a do/end syntax) that would be your block. If you have worked with statements like these, then you have worked with blocks. One thing that is important to mention with blocks would be the yield statement. If you have a method and then make a block as a parameter, you can use yield inside of the block to call that! Confusing? Let's see an example.

def cheer
puts "Hip hip"
yield
puts "Hip hip"
yield
end

cheer {puts "Hooray!"}

The Result:
Hip hip
Hooray!
Hip hip
Hooray!

You can see that yield will put whatever is inside of the block. You can add parameters to the yield statements too.
